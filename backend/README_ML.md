# 🤖 Machine Learning для предсказания цен на квартиры

## 📋 Что сделано:

### 1. **train_model.py** - Скрипт обучения модели
Создает и обучает ML модель на данных из `krisha_clean.csv`:
- Очистка и preprocessing данных
- Feature engineering (15 признаков)
- Обучение 3 моделей: Ridge, Random Forest, Gradient Boosting
- Сохранение лучшей модели в `model.pkl`

### 2. **ml_predictor.py** - Класс для предсказаний
- Загрузка обученной модели
- Preprocessing входных данных
- Предсказание цены с диапазоном

### 3. **main.py** - API с ML интеграцией
- Автоматическая загрузка ML модели при старте
- Использование ML для предсказаний (если модель есть)
- Fallback на простой расчет (если модели нет)

## 📊 Признаки модели (Features):

1. **Базовые:**
   - `room_count` - количество комнат
   - `quadrature` - площадь (м²)
   - `floor` - этаж
   - `year` - год постройки
   - `ceiling` - высота потолков

2. **Вычисляемые:**
   - `age` - возраст здания (2024 - year)
   - `rooms_per_m2` - комнат на м²

3. **Бинарные (0/1):**
   - `has_parking` - наличие парковки
   - `has_furniture` - наличие мебели
   - `has_balcony` - наличие балкона
   - `has_glazed_balcony` - наличие застекленного балкона
   - `has_internet` - наличие интернета
   - `is_first_floor` - первый этаж

4. **Региональные:**
   - `region_avg_price_m2` - средняя цена за м² в районе
   - `region_encoded` - закодированный район

## ❓ FAQ:

**Q: Почему модель не загружается?**
A: Запустите `python train_model.py` для создания модели.

**Q: Можно ли использовать без ML?**
A: Да, API автоматически использует простой расчет, если модели нет.

**Q: Как часто переобучать модель?**
A: При появлении новых данных или раз в месяц.

**Q: Сколько времени занимает обучение?**
A: 1-3 минуты на обычном ноутбуке.
